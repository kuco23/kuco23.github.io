Python System


  • Classes

    • Class initializes an instance throught the __init__ method and stores the instance into the first argument of __init__(instance) function. Later on you can pass the instance around the other functions in class, note that any method called as instance.method() is getting instance passed as a first argument, so instance.method() == type(instance).method(instance). The standard way to save a new instance created through class is by calling it self
    self represents an instance being used or created localy in class methods

    calling the method through instance means the method is bound to instance of class type(instance), calling the method through class (type(instance)) means the method is bound to class of the instance.

    A way to make a method unbound to its instance while still being able to access it through an instance is by using a decorator @staticmethod before defining function in the class.
    In that case this is true: instance.method() == type(instance).method.

    A common decorator used is @classmethod, which makes a method called through instance bound to its class rather than the instance.
    In that case this is true: instance.method() == type(instance).method(type(instance)).

    Instance of a class = an object initialized through said class.
    Type of instance = a class from which the instance was created
    a = className() --> a is an instance of className
    isintance(a, className) --> returns True

    isinstance(instance, type) == (type(instance) == type)
    isinstance is better as it checks whether instance is an instance of any superclass of type

  • Magic Methods

    • Here self will represent an instance of a class in which the method using it is defined
    Magic Methods are methods that are used to being called on instances from outside the class.
    __init__(self) -- class constructor: is called when the class is called and creates the instance of its class
    __repr__(self) -- class representator: is called when an instance is called by itself and must return string
    __str__(self) -- class string converter: is called when trying to convert object to string (str(self))
    __mul__(self, other) -- multiplication: defines the result of self * other
    __rmul__(self, other) -- rightsided multipication: usually used when defining multiplication for different types of instances
    __pow__(self, n) -- power: defines self^n
    __getitem__(self, index) -- defines what to return when calling self[index]
    __eq__(self, other) -- equality: defines what to return when comparing self == other
    __call__(self) -- defines what to return when calling self - self()
    __name__(self) -- returns the name of file self class is written in, if it's the same as the one you're calling it from it returns 'main'

  • Subclassing

    • Subclassing an existing class to a new class means this:
    class myNewSubClass(SuperClass)
    This takes SuperClass' methods and applies it to myNewSubClass
    (Now myNewSubClass derives from SuperClass; also called inheritance)
    Every method name and the names of instance variables are the same as in SuperClass.

    This is useful when trying to override some of the methods in SuperClass
    This is done by defining a method in myNewSubClass with the same name as the one you wish to override
    This can mess the code if the arguments that instances depend on change

    When overriding a method somethimes you will wish to keep some of the SuperClass code
    This is done by calling super().method_to_replace(*args)
    self is already called with super, so do not put it in *args
    The old way: super(myNewSubClass, self).method

    If you do not rename or create any new methods in SubClass while SuperClassing SuperClass, then:
    Subclass == SuperClass

  • Args & Kwargs

    • *args

      • args stands for arguments and is a common name in practice, otherwise only the asterix is necessary (eg. *something). Imagine you do not have any need for the arguments in the function, and are just planning to pass them to another function, or maybe you don't know how many arguments your function will take and you just need the first two or something.
      This is where *args comes in. By writing
      def function(*args): return args
      The args variable is assigned every argument, passed to a function, as a tuple function(1,2,3) will return tuple(1,2,3)
      Outside functions a = 1,2,3 also returns a tuple (1,2,3)

      You could imagine that *args eats up every value (that is left) from an iterable assigned to it. This can also be useful for dealing with stuff outside functions. Eg. *var1, var2 = (1,2,3), which save (1,2) in var1 and 3 in var2. This is called unpacking and is discussed in the Tuple section

    • **kwargs

      • Kwargs stands for Keyworded arguments, which basically means that the arguments you pass in a function have a key infront of them (eg. argument1 = 'myKwarg'). As with *args, kwargs name is optional and only ** is required

      So **kwargs eats up all the keyworder arguments, and saves them in a dictionary
      def function(**kwargs): return kwargs
      if called with
      function(a=10, b=30) will return {'a':10, 'b':30}

  • Handling Exceptions

    • Exceptions are Exceptional events, and should not be confused with Errors
    Exceptions can be handled, while Errors should crash the program
    (memory overflow causes an error, division by zero causes an Exception)

    In Python exceptions are handled with try-except-finally blocks
    try: 1/0 except ZeroDivisionError as e: print(e)
    The code the code will print the error without stopping the execution of a program

    This might be useful, if a certain exception is expected to appear often in a program
    and is not critical to the rest of execution, so there is no need for it to cause
    the program to crash, and rather be handled.

    • try block

      • try is the beginning of any exception handling and must be written first
      It tries to execute the code block in it and passes any exception that might happen
      to exception handlers that are defined within:

    • except block

      • except ExceptionType as e: doStuff
      If the exception that occured in the try block matches the ExceptionType in
      the except block, then doStuff gets executed
      If exceptions do not match Python continues on trying to match the exception
      to other latter except blocks, until the end of exceptions
      If it comes to the end of the block without handling the exception, exception causes
      the program to crash, as it would normally.
      But what if you need something to be done (notorious example: file closed), no matter
      what happens within the try-except code. That brings us to:

    • finally block

      • The finally block (finally: code) executes no matter what
      This means that if the program should crash, because exception occured, not crash, because
      the exception was handled, or if handling of the exception was not necessary, the finally
      code will always execute.
      (usually used for closing a file, though python provides a better way to do this, which will be discussed later).

    • Exception Types

      • Exception -- SuperClass of every exception (except Exception handles every exception)
      StopIteration -- Raised when the next() method of an iterator does not point to any object.
      ArithmeticError -- Base class for all errors that occur for numeric calculation.
      OverflowError -- Raised when a calculation exceeds maximum limit for a numeric type.
      ZeroDivisionError -- Raised when dividing by zero occures
      KeyboardInterrupt -- Raised when program is interrupted by user (usually with Ctrl-C)

    • Custom Exceptions

      • You can define custom exceptions by SuperClassing base class to all exceptions : Exception
      class MyException(Exception)
      Every argument passed to Exception __init__ with super() will be written in red when exception ocurres
      You raise your exception (or others) by calling raise MyException(args)

  • Tuples

    • Tuples are like lists except they are more immutable
    They should be used for containing fixed amount of stuff
    They can be iterated through and support indexing

    The most important feature of tuples is unpacking
    Unpacking basically means that by assigning multiple variables to one tuple is allowed
    if number of variables <= len(tuple) (< if last argument is started by *)
    var1, var2, var3 = (1,2,3) or var1, *var2 = (1,2,3)

    If you want to save some values into a variable and dispose of others you can do:
    a, b, *_ = (1,2,3,4,5,6)

    The most useful way to use tuples though, is by using them to iterate through
    by assigning two values every iteration
    for i, j in [(1,2), (2,3), (3,4)]: print(i,j)
    This will return (1,2), (2,3), (3,4)
    So imagine if you have multiple iterators, you would like to iterate through simultaniously
    (note that iterators might not support indexing)
    This is where zip comes in. zip(iter1, ..., iterN) creates a new generator, you can iterate through
    with N variable assignments every iteration
    Basically zip(iter1, iter2) = ((iter1[0], iter2[0]), ..., (iter1[N], iter2[N])), N = min(len(iter1)-1, len(iter2)-1)
    note that this is simplified and that zip creates a generator, which
    returns objects only when iterated through and will be emptied by the end of an iteration
    Also iter1 and iter2 might not support indexing, being generators themselves

  • Generators

    • Generator is an iterable that does not store objects, but rather creates them on the fly.
    There are two methods (that i know of) that create generators.
    Generators cannot be created as instances from a class
    Generators support the method next(var_generator) that returns the next value of var_generator
    If a generator is not infinite, it can be emptied (reaches its end)
    Once you go forward you can never go back
    Generators support tuple or list conversion, which means they iterate themselves into list/tuple

    • Yield

      • Imagine you do not need a all values stored in a list, you just need
      to occasionaly generate some with a complicated code
      This is where yield comes in.
      By replacing return with yield inside a function, usually yielding value by the end
      of a single iteration, and saving called function in a variable
      var = myGenerator() you create a generator

    • Tuple one-line Loop

      • The fastest way to create a simple generator is with an one-line loop
      myGen = (i for i in range(10))
      This creates a generetor that yields values 0 through 9

Java Basics

  • Basic Syntax

    • Accessibility prefixes

      • public --> variable, function can be accessed from everywhere,
      meaning outside class, inside class, main subclasses (when superclassing a class in local file)
      or package subclasses (when superclassing a class in imported file)

      protected --> variable, function can be accessed from package,
      in which it was declared + subclasses or package members

      no modifier / default --> variable, function can be used in package, in which it was declared

      private --> variable, function can be used in class, in which it was declared

      Class Package Subclass (same package) Subclass (diff package) World
      public - - - - -
      protected - - - -
      no modifier - - -
      private -

    • Value Type Prefixes

      • Used to declare variables or function's return type
      void --> declaration function doesn't return anything (used only on functions)
      int --> used to declare primitive-data-type integer of 32-bit signed integers (max +- 2^31)
      long --> used to store 64-bit signed integers (max +- 2^63)
      double --> declaration of primitive-data-type of abstract class Double, meaning double-precision (64-bit) numeric decimal values
      float --> same as double, except values are of single precision (32-bit)
      short --> used to store max +- 2^15 (16-bit signed) integer
      byte --> used to store max +- 2^7 (8-bit signed) integer
      boolean --> used to store binary values true and false
      String --> used to store strings (they are immutable type - cannot be changed, use StringBuilder for that)
      char --> used to store single letters
      <data-type>[] --> used to store a list of data-type types

      Every numeric type primitive-data-type (used to perform operations like +-*/ on) has an abstract class
      used to store methods that can be applied to primitive-types
      (eg. classes: Integer, Double, Float, Long)

    • Other Type Prefixes

      • static --> declaration of variables, functions, methods that are used by instances, but
      cannot be used on instances, because all instance share the same static methods.
      Static method belongs only to class containing it.
      Static functions can be accessed outside the class without calling the constructor

  • Data Types Info

    • Integer

      • Subclass of an abstract class Number, representing 32-bit integers between -2 147 483 64 and 2147 483 647.
      Integer.toString(int/Integer var) --> returns a String reperesentation of input var
      Integer.parseInt(String) --> returns an Integer representation of String (eg. "200")

    • Long

      • Used to declare 64-bit signed integers (from -2^63 to 2^63-1). When assigning long values to variable the L must be put at the end of number value or (long) at the beginning. Default data-type assigned to integers (numeric types without decimal value explicitly written) is int. (eg. long var = 10000000000L).

    • Double

      • Subclass of an abstract class Number, representing double precision 64-bit IEEE standard decimals
      Double.toString(double/Double var) --> returns a String representation of input var
      eg. 2.114124134

    • Float

      • Subclass of an abstract class Number, representing double precision 32-bit decimals

    • Short

    • Byte

      • Stores an 8-bit value integer (integers between -128 and 127 [-2^7, 2^7-1])

    • Boolean

      • Stores one bit of information, true or false

    • Character

      • Primitive data-type = char
      Used to save a single letter into a variable
      Do not use (int) to convert (char) var into numeric value of char
      Character.toString(char) --> returns a String representation of a char
      Character.getNumericValue(char var) --> returns a numeric value of char var
      Character.isDigit(char) --> reuturns true if input char is a char representation of an integer (eg. "200")

    • String

      • String is not a primitive-data-type, but a class, so it can be instantiated as
      String var = new String("string") or
      String var = new String(char[] var)

      StringVar.length() --> returns length of StringVar
      StringVar.charAt(int index) --> gets the char at StrigVar[index]
      StringVar.split("sign") --> returns String[], that has elements of a StringVar split on every "sign" occurance
      (eg. String[] = "stringastring2".split("ab") --> returns ["string", "string2"])
      String.join("sign", String[]) --> returns a String that is a glued-up input String[] with "sign" in the middle []String elements
      StringVar.substring(int begin, int end) --> returns sliced String from begin index letter to (not including) int end index letter

    • StringBuilder

      • Used for String modification. StringBuilder class has every method of String class
      StrinBuilder var = new StringBuilder(String) --> initializing StringBuilder object
      var.setCharAt(int index, char ch) --> changes var[index] to ch
      var.append(String) --> adds String to the end of var

    • File Methods & Objects

      • FileInputStream --> used for reading from files
      FileInputStream input = new FileInputStream(StringFileName);
      supports method .read(), which generates letters from start of file to finish, and when it reaches the end, it returns -1 (It generates them by moving forward one letter every time read() is called).
      Data generated from instance method .read() produces int type data, that can be converted to char using (char)

      FileOutputStream --> used to output stuff in files
      FileOutputStream output = new FileOutputStream(StringFileName);

      FileReader --> used for reading from files
      FileReader reader = new FileReader(new File('fileName'));
      The FileReader supports instance method .read(char[] var), which fills char[] var with characters read from file

      PrintWriter --> used to print/output stuff in its instance file
      PrintWriter writer = new PrintWriter(String fileName);
      acts in a similar way that System.out.<method> does, the differernce being that writer.println("text") prints in fileName

    • Empty File Text Into String Example

      • Java

      import java.io.*;

      public class MyFiles {
        public static void main(String[] arg) throws IOException {

          FileInputStream = null;
          File myFile = null;
          FileReader reader = null;

          int n = 0;
          char[] FileContent = null;

          try {
            input = new FileInputStream("MyFileName");
            while (input.read() != -1) {n += 1;}
            } finally {input.close();}
          // we've used FileInputStream to determine
          // the length of text in "MyFileName"

          try {
           MyFile = new File("MyFileName");
           reader = new FileReader(MyFile);
           FileContent = new char[n];
           reader.read(FileContent);
           } finally {reader.close();}
          String FileText = new String(FileContent);
          // now we have file text saved in our FileText variable
          }
        }

  • Html Stacking Order

    • Understanding what gets put infront and what behind elements

    • The stacking context's root element (the <html> element usually)
    • Positioned elements (and their children) with negative z-index values (higher values are stacked in front of lower values; elements with the same value are stacked according to appearance in the HTML)
    • Non-positioned elements (ordered by appearance in the HTML)
    • Positioned elements (and their children) with a z-index value of auto (ordered by appearance in the HTML)
    • Positioned elements (and their children) with positive z-index values (higher values are stacked in front of lower values; elements with the same value are stacked according to appearance in the HTML)
