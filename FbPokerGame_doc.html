FbPokerGame documetation

Intro

This is the documetation for the PlayMessinger app that allows playing poker on messinger. It depends on the fbchat module, which is used for I/O.

Basics

The basis of this application is the Dealer account, which can be set to any facebook account. It is recommended for the application user to set up a new account, reserved for the Dealer, or use any other account which is not being used by anyone. The Dealer's account will be automated to send game information (eg. name of the player whose name is expected) to users and group threads.
Any player that is removed from the group will be added back by the Dealer. If the Dealer is removed from the table the concequences shall be dire.

Player Rules

New Player introduction

Any player that is playing a round for the first time will get 4000 as a starting amount, that will be logged in his datafile in the database directory.

Player Money

The player's money and the money that he plays with are seperated. The money that player plays with is entirely within the game and can be withdrawn almost anytime with this statement.
The outside money is seen as the money you can use to buy in to play on other tables, that require a certain amount of money to participate in.
If player's outside money is less than game buy-in of 1000, player can buy in with less than 1000 to play in the game, else he must put 1000 of his outside money in game to participate.
The table money is seen as the money you currently play with on a certain table. Any player who hasnt lost all the table money (his table money is 0) is considered participating. It is not added to the outside money until a player withdraws it or Dealer is removed from the table, after which the game is aborted and the money put in the pot lost.

Player Statements

Game Manipulation Statements

::init Adds the table to the recognized game tables and starts the series of rounds if possible (number of players with money is adequate). ::lastround Toggles the last round status.
Every round is pre-configured to begin another after it ends, so with this it is possible to interrupt the following round's execution. The statement must be sent from a participating player as those not participating are considered not in the game and can only observe it. After ::lastround is sent to the table from a participating player it ends the game after the round is finished. The ::lastround command can be canceled (toggled) with writing ::lastround again. After the game has ended it can be restarted by sending ::init to the group
After the game has come to an end player's money still remains in the game and has to be withdrawn!

Player Group Statements

Statements that are written within a group to get information or affect player's state. ::money Shows the money player has left on the current table by sending it to the group from which the message was sent from. ::refill Refills user's money to the table buy-in of 1000 and subtracts the difference from outside money. Works only if the player is folded, not participaating or round is not being played. ::withdraw Withdraws player's money from the table and adds it into the player's outside money. Works only if a player is folded, not participaing in round or a round is not being played.

Player Private Statementes

Statements that are written to a Dealer's account directly, requesting info or an action to be taken. show money Shows the outside money user has left by sending it to the user's account. gimme moneyz Requests the money of 100 from Dealer which can be given every 4 hours.
