Pridobitev seznamov sumljivih kripto naslovov

bitcoinabuse (https://www.bitcoinabuse.com)

  • pridobivajo podatke preko prijav uporabnikov
  • ne brišejo prijav

in order to protect the integrity of the BitcoinAbuse database we will not remove or edit any reports.

API

Za uporabo zahteva API token, ki je z računom zlahka pridobljen.

API omogoča

  • Prijavo danega naslova

  • Preverjanje danega naslova

  • Prenos zgodovine prijav (v csv)

    • 1 dan
    • 1 teden
    • "forever"

The "forever" file is updated monthly on the 15th of the month.

All updates occur between 2am-3am UTC.

Izgradnja lastne baze

Ker se prijave ne brišejo, za posodobitev baze potrebujemo le nove prijave.

Bazo tako lahko ohranjamo z

  • Prenos "forever" baze 15.xx.xxxx
  • Vsak dan prenos še dnevne zgodovine (ob 3am UTC)

Ker API omogoča oboje, je to enostavno avtomatizirano.

Prijave

Vsaka prijava vsebuje

  • id

  • address (bitcoin naslov)

  • abuse_type_id

    1. ransomware
    2. darknet market
    3. bitcoin tumbler
    4. blackmail scam
    5. sextortion
    6. other

  • abuser (večinoma mail)

  • description (večinoma sporočilo abuser-ja)

  • from_country

  • created_at (čas)

 

BitcoinWhosWho (https://bitcoinwhoswho.com)

Pridobivajo prijave preko uporabnikov. API omogoča pregled prijav za vnešen naslov in do 100 klicev dnevno.

Klic vrne json s poljem "status" vrednosti "error"/"success". Če je status "success", vsebuje še seznam prijav s polji

  • reported_date
  • scam_category
  • scam_details
  • image_large / image_small (povezava do priložene slike)

Polja so lahko prazna ("")

CryptoScamDB (https://cryptoscamdb.org)

Hrani zlorabe večih kriptovalut, večina pa je ETH.

Podatke pridobivajo sami, ter prek prijav uporabnikov.

Za API ne potrebujemo ključa in omogoča

  • Dobiti status vnešenega naslova/domene/ENS
  • Dobiti celotno bazo naslovov, skupaj z prijavami
  • Dobiti whitelistano bazo naslovov

 

Podatki, dobljeni s klicem https://api.cryptoscamdb.org/v1/addresses vsebujejo

  • naslov
  • seznam prijav asociiranih z naslovom

 

Prijave vsebujejo tip prijave, ki je vrednosti

  • Phishing
  • Scam
  • Scamming
  • Fake ICO
  • None

Omejevalni Ukrepi

 

EEAS seznam

Identifikator entitete je po vsej verjetnosti //sanctionEntity/@logicalId.

Primeri, ko se ta podvaja

  • (Zahid Ahmadzai, Mohammad Zahid, Jan Agha Ahmadzai), (Zahid Mohammad)
  • Abdul Wali Seddiqi, Wali Qari Abdul
  • Attiqullah Akhund, Akhund, Attiqullah

Vsebuje /regulation/@publicationDate, torej datum objave.

Glavne lastnosti entitet

  • Tip (subjectType@code ('enterprise' ali 'person'))
  • Ime (nameAlias@wholeName)
  • Spol (nameAlias@gender)
  • Državljanstvo (citizenship@countryIso2Code)
  • Naslov
  • Država rojstva (birthdate@countryIso2Code)
  • Leto rojstva (birthdate@year)

To bi bili glavni kandidati za filtracijo pri iskanju osebe.

Posebnosti

  • Osnovni podatki so hranjeni kot atributi
  • Skoraj vedno je atribut označen za praznega kot attr="", pri redkih pa lahko manjka

 

OZN seznam

Enoličen identifikator vnosa (individual / alias) je <REFERENCE_NUMBER>.

Vsebuje <LAST_DAY_UPDATED> značko (tudi <LISTED_ON> za datum prvega vnosa).

Glavne lastnosti vnosov

  • Tip (individual / entity)
  • Ime (first, second, third, fourth name)
  • Spol (GENDER)
  • Datum rojstva (INDIVIDUAL_DATE_OF_BIRTH/DATE)
  • Naslov (INDIVIDUAL_ADDRESS/COUNTRY)

Posebnosti

  • najprej loči INDIVIDUAL ter ENTITY znački.
  • Značke nimajo atributov
  • standarden zapis prazne značke je <tag /> (lahko manjka)

SDN seznam

Identifikator vnosa je <uid>, ne vsebuje podatka o datumu vnosa podatkov.

Glavne lastnosti entitet

  • Tip (sdnType) (Aircraft, Individual, Entity, Vessel)
  • Ime (firstname, lastname)
  • Država / mesto
  • Program (določene vrednosti)

Posebnosti

  • lastname je vedno prisotno ime vnosa (tudi če to ni oseba)

Algoritmi za iskanje po seznamih

Python paketi za fuzzy-matching

  • textdistance
  • Fuzzy
  • fuzzywuzzy

Jaro-Winkler

  • Jaro - primerja sorodnost nizov z ujemanjem črk v neki okolici ter štetjem transpozicij
  • Winkler - prioritizira ujemanja na začetku
  • Zahtevnost je O(n^2)

Soundex

  • Kodira besedo v sojo fonetično reprezentacijo
  • Problem: namenjen je le za angleščino
  • Nadgradnja Metaphone: podpira še nekatere jezike a ne vseh

 

Levenshtein

Problem: Je O(n^2) a povprečno res počasnejši od Jaro-Winkler

Jaccard

  • Primerja ujemanje znakov ne glede na vrstni red
  • Zelo hiter

Z njim preverimo, če sta niza preveč različna.
